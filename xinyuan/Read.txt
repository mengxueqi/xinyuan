可以把它当成一个“产业情报监控系统”来设计，而不是单纯的新闻聚合器。核心目标不是“收集信息”，而是“发现变化、判断重要性、及时提醒”。

一、先定义这个程序要解决什么问题

建议聚焦 3 类用户价值：

1. 追踪“最新资讯”
包括融资、合作、扩产、产品发布、监管动态、技术突破、招聘变化、专利动作。

2. 追踪“变化”
不是只看新文章，而是发现企业状态变化，比如：
- 官网新增产线
- 招聘岗位从研发转向生产/商务
- 管理层变动
- 临床/注册/审批状态变化
- 产能、客户、合作伙伴变化

3. 输出“可行动情报”
最终不是给用户一堆链接，而是告诉他：
- 哪家公司最近值得关注
- 为什么值得关注
- 和上周/上月相比发生了什么变化

二、MVP 最小可行版本建议

第一版不要做太大，先只做“少量公司 + 少量信息源 + 每日更新 + 变化提醒”。

MVP 我建议包含 6 个模块：

1. 监控对象管理模块
用来维护你要追踪的公司和关键词。

最少字段：
- 公司名
- 官网
- 细分赛道
- 地区
- 关键词
- 竞品/别名

为什么它必须先有：
因为后面所有抓取、分类、提醒都依赖这份“监控名单”。

2. 数据采集模块
MVP 阶段先抓最稳定、最容易拿到的源：

优先级建议：
- 公司官网新闻页
- 公司博客/公告页
- Google News / Bing News 结果页
- LinkedIn / 招聘页
- 行业媒体 RSS 或文章列表页
- 监管机构公告页

采集方式：
- RSS 优先
- 静态网页抓取其次
- 动态页面最后再做

MVP 不建议一开始就接太多平台 API，否则维护成本高。

3. 信息清洗与去重模块
抓回来的内容会很乱，必须做基本处理：

最少能力：
- 去重
- 提取标题、发布时间、来源、链接、正文摘要
- 识别提到的公司
- 标签分类

标签可先做成固定枚举：
- 融资
- 合作
- 扩产
- 招聘
- 监管
- 技术发布
- 专利/IP
- 市场/销售
- 高管变动

4. 变化检测模块
这是这个产品最有价值的部分。

MVP 只做 3 种变化就够了：
- 新增事件：今天第一次出现的资讯
- 页面内容变化：公司官网某页面内容更新
- 招聘变化：新增岗位/岗位方向变化

实现思路：
- 保存上一次抓取结果
- 本次抓取和上次做 diff
- 只输出“有意义的变化”

例如：
- “官网新增 GMP manufacturing 页面”
- “招聘新增 process engineer / QA director”
- “新闻页新增与某 pharma 公司的合作公告”

5. 情报摘要与评分模块
不是每条信息都重要，所以需要简单排序。

MVP 可以先做规则评分，不用一开始就上复杂模型。

示例评分维度：
- 是否来自官网/监管源
- 是否提到融资金额
- 是否提到产能、工厂、商业化
- 是否涉及大客户/大药企合作
- 是否是连续多次出现的强信号

输出格式：
- 事件摘要
- 变化说明
- 重要性评分
- 建议关注原因

6. 通知与展示模块
MVP 不需要复杂前端，先能“看”和“收提醒”就够了。

推荐最小方案：
- 一个简单 Web dashboard
- 每日/每周邮件摘要
- 企业详情页展示时间线
- “今日新增变化”列表

如果资源有限，甚至可以先只做：
- 后台定时跑
- 结果写到数据库
- 每天生成一份 markdown/html 报告

三、MVP 之外先不要急着做的东西

这些很容易让项目变复杂，建议第二阶段再上：
- 全自动行业知识图谱
- 高级舆情分析
- 多语言全量翻译
- 复杂权限系统
- 很重的实时流式架构
- 大而全的数据源接入

四、推荐的 MVP 技术架构

如果你想快速落地，建议走这条路线：

- 后端：Python
- 抓取：requests + BeautifulSoup + feedparser，必要时 Playwright
- 定时任务：APScheduler / Celery
- 数据库：PostgreSQL
- 搜索：先用 PostgreSQL 全文搜索，后续再考虑 Elasticsearch
- 前端：React 或者直接先用 Django Admin / 简单模板页
- AI 能力：
  - 文章摘要
  - 事件分类
  - 变化描述生成
  - 企业实体识别辅助

数据库核心表建议：
- companies
- sources
- raw_documents
- events
- change_logs
- job_snapshots
- alerts

五、一个很实用的 MVP 工作流

可以按这个流程跑：

1. 配置一批目标公司
2. 每天定时抓取官网、新闻、招聘页
3. 清洗和去重
4. 对比历史快照，识别变化
5. 把结果分类、打分、生成摘要
6. 输出到 dashboard 和日报

六、建议你先限定追踪范围

不然很容易做散。

MVP 最好限定为：
- 只追踪 30-50 家公司
- 只覆盖 3-5 类信号
- 只做每日更新
- 只做中英文或单一语言
- 先聚焦一个垂直方向

例如可以先专注：
- CDMO/生物制造公司
- 合成生物学企业
- 细胞治疗制造企业
- 发酵与蛋白制造平台公司

七、如果让我帮你拆 MVP，我会这样定义

最低可上线版本：
- 公司库管理
- 多源抓取
- 去重与分类
- 历史快照保存
- 变化检测
- 日报/提醒

这 6 个模块就足够验证价值了。
